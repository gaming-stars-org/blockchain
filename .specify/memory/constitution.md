# Gaming Start Constitution

## Core Principles

### I. Simplicity First
Prefer simple, maintainable solutions over complex abstractions.

### II. Clear Interfaces
Any module or API must have a clear input/output contract and basic usage documentation.

### III. Test Coverage
New behavior must include tests or a documented reason why tests are not feasible.

### IV. Safe Changes
Breaking changes require explicit note in the PR/commit and a migration path when needed.

### V. Visibility
Runtime errors must be logged with enough context to diagnose failures.

## Constraints

Use the existing project stack and patterns unless a change is justified in writing.

## Workflow

All code changes should be reviewed, and CI checks should pass before merge.

## Governance

This constitution is the baseline for engineering decisions. Amendments require a documented rationale and approval by project maintainers.
